Morkulla - House Calculator

Welcome to our Expo page!

We have developed a product for Devolv, a company that helps you build your dream home. For more information about Devolv, check out Devolv's website. Our product is a web application that answers questions about the costs of house construction. First, it asks you questions about your house construction project. Then, it presents graphs and tables regarding the cost of the project and bank loan payments. The prices in the industry vary a lot. Therefore, we allow for full customization of the costs.
We used Scrum as our project methodology, and Git as our version management tool. The product itself is developed using mainly the Svelte framework and Typescript.

Sprint summaries

Sprint 1

The project began here; our goals for this sprint were to implement basic site navigation, set up charts, and finish the UX design of the website with approval from Devolve.
This went mostly well, although we did not get complete approval of the UX design. We also discovered problems with our communication with Devolv.

Sprint 2

For the second week of the project we determined that the goals were to improve communications with Devolv, improve the UX design so that it could be approved, refactor code, and to create a navigable dummy site.
The most important goal of these was to improve communications, which was achieved by compiling a list of what information we needed but did not have from Devolve in order to approve mutual understanding.
Moreover, the design was not fully approved again. With the design aside, everything went well.

Sprint 3

Our goal for sprint 3 was to finish the website in such a capacity that it would work for its intended use by customers. This went reasonably well in itself, but Devolv wanted to change some earlier design decisions, which had us redo some work. We could not finish the entire base for the website because of the design changes.
We also began to have scarse work, there was not much more to do that could be parallelised for eight developers which made work assignment difficult.

Sprint 4

For the fourth and final sprint, we decided that our goals would be to fix as many bugs as we could and to prepare for presenting the project. The final version of the project is the one linked earlier on this page, please check it out!
